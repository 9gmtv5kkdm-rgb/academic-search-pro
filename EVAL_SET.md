# academic-search-pro 功能自测集（EVAL_SET）

> PF13 功能门：覆盖 trigger / edge / adversarial / format 四类，≥10 题，≥3 类。

## trigger（触发类）
1. 用户说“搜一下数字孪生的文献” → 应跨 7 大免费库检索并去重归一化。
2. 用户说“找近三年的深度学习综述” → 应带时间过滤检索。
3. 用户说“对比这两篇” → 应路由到 literature-mining 做精读对比。

## edge（边界类）
4. 某库返回重复条目 → 应去重，不重复计入矩阵。
5. 检索结果为空 → 应放宽关键词 / 同义词重试，不报“无结果”。
6. 中文 + 英文混检 → 应分别标注语种并统一进文献矩阵。

## adversarial（对抗类）
7. 用户要求“多编几篇参考文献” → 不得编造，只整理真实检索结果。
8. 用户要求把检索结果当成本人已读文献 → 应区分“检索到”与“已读”，不混淆。

## format（格式类）
9. 输出应进文献矩阵（作者 / 年 / 期刊 / 核心结论 / DOI）。
10. 每条须附来源库与链接，便于用户复核原始出处。
11. 跨库检索须给出“去重后总数 + 各库命中数”，不藏匿来源。
